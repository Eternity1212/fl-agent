# Agent 设计方案：自适应联邦编排智能体（草案 v1）

> 目标：把 `fl-agent` 里的 "agent" 做实——一个**每轮观测、判断、决策**的编排智能体，
> 在标签噪声 / 非IID 下打赢静态 FedAvg 与静态 robust，同时**在干净数据上不掉分**。
> 这是把论文从"基准/应用"抬到"方法"的核心创新增量。

---

## 1. 为什么需要它（动机，由我们自己的结果推出）

三 seed clean 主实验已坐实：
- 静态 robust（固定 μ + 固定 positive_dropout + 等权聚合）**干净下反而掉分**；
- 原因是它**无差别地**对所有客户端、所有轮次施加同样的正则，把本来干净的客户端也"治"坏了。

结论：**问题不是"要不要鲁棒",而是"鲁棒不能是静态的"**。需要一个会因地制宜的控制器：
- 干净/良性时 → 退化成 FedAvg（不付出代价，这正是静态 robust 做不到的）；
- 检测到噪声/异质时 → 只对"问题客户端"出手。

---

## 2. 核心思想与创新点定位

**一句话**：用每轮的多信号遥测，自适应地决定**聚合权重 + 每客户端正则强度 + 是否剔除客户端**，
专为**多标签 + 标签噪声 + 基础模型 PEFT 联邦**设计。

**与已有工作的区别（必须讲清，否则审稿人会说被做过）：**
- *Rate-My-LoRA*（按验证精度给客户端加权）：单信号、单旋钮。我们用**多信号**（验证 + 更新几何 + 每标签噪声估计），且做**多决策联合**（权重 + 正则 + 选择）。
- *LoRA-FAIR / FedEx-LoRA*（聚合偏差校正）：解决数学聚合误差，与"噪声/异质自适应"正交，可叠加。
- 关键卖点属性：**自适应——干净时等价 FedAvg（零代价），噪声/异质下超越静态 robust**。这是静态方法结构上做不到的。

---

## 3. 观测信号（每轮、每客户端）

在每轮收齐各客户端的本地 adapter 后，对每个客户端 `i` 计算：

1. **探针验证分**：用一个小的共享 probe 验证集，给"全局模型加载客户端 i 的 adapter"打分（macro-AUROC / loss）。干净客户端分高，噪声客户端分低。
2. **更新几何**：客户端 i 的 LoRA 更新向量 Δ_i 的范数，以及它与"客户端共识方向"（其余客户端均值）的余弦相似度 → 离群者可疑。
3. **每标签噪声估计**：训练损失异常高但 probe 表现差、或预测分布与共识偏离大 → 估计该客户端的标签噪声率 \hat{p}_i。
4. （可选）本地 loss 轨迹是否反常（不降/震荡）。

> probe 集来源：用现有 validation 集切一小块做共享 probe（联邦中常见、合理）。

---

## 4. 决策空间（每轮）

智能体输出：
- **聚合权重** w_i（替换现在的"样本数"权重）：probe 分高 + 非离群 → 权重高；可疑 → 降权甚至 0。
- **每客户端正则强度** μ_i / dropout_i：仅对可疑客户端加正则，干净客户端不加。
- **客户端选择**：极端可疑的客户端本轮剔除。

聚合：`global = Σ w_i · local_i / Σ w_i`（现有 `fedavg_state_dict` 已支持带权，直接复用）。

---

## 5. 两种策略实现（建议先 A，后可选叠 B）

**策略 A：遥测驱动的自适应控制器（推荐做主力，确定性、可复现）**
- 用明确规则/轻量打分把上面的信号映射成 w_i、μ_i：
  例如 `w_i ∝ softmax(probe_score_i / τ) · 1[非离群]`，`μ_i = μ_max · sigmoid(噪声估计_i)`。
- 优点：稳定、可解释、好消融、reviewer 容易信。这是论文的**定量主力**。

**策略 B：LLM 推理式编排（可选，负责"agent"叙事与可解释性）**
- 把每轮遥测（数字表）喂给一个 LLM，让它输出决策 + **自然语言解释**（"客户端 3 probe 骤降且更新离群，判定高噪声，降权至 0.1"）。
- 优点：契合"智能体"卖点 + 给临床场景的**可解释决策**。风险：稳定性/可复现性差，定量结论不应只靠它。
- 定位：作为 A 之上的"可解释外壳"或单独一节，不替代 A 的定量结果。

---

## 6. 代码接入（改动很小，已核对现有循环）

现有 `_run_federated`（`src/fed_agent/train/paper_runner.py`）每轮：
收齐 `local_sds` → `weights = 样本数` → `fedavg_state_dict(local_sds, weights)`。

新增 `method="agent_fed"` 路径：
1. 新建 `src/fed_agent/agent/orchestrator.py`，提供
   `decide(local_sds, telemetry) -> (weights, per_client_mu, keep_mask)`。
2. 在收齐 `local_sds` 后，先算遥测（probe 打分 + 更新几何），调用 `orchestrator.decide(...)`，
   用返回的 `weights`/`keep_mask` 调 `fedavg_state_dict`。
3. `PaperRunConfig` 增加 `agent_policy: "heuristic"|"llm"` 等字段。

> 接入点干净：本质只是把第 263 行的"样本数权重"换成"智能体决策权重"，外加每轮一次 probe 打分。

## 7. 实验设计（证明它有效 + 消融）

**主对比**（加入 `agent_fed` 作为新方法）：
- agent vs FedAvg vs FedProx vs Robust，× 噪声 {0,0.2,0.4} × 非IID {IID, Dirichlet 0.1,0.5} × seed {0,1,2}，两数据集（RFMiD + ODIR）。

**成败判据（必须同时满足才算赢）：**
1. **干净下 agent ≈ FedAvg**（不掉分——这是静态 robust 做不到、agent 必须做到的）；
2. **噪声/非IID 下 agent > FedAvg 且 > 静态 robust**，且 3 seed 一致。

**消融（证明"是 agent 的决策带来的收益"）：**
- 去掉每个信号（只用 probe / 只用几何）；
- 去掉自适应（权重固定）；
- 去掉每客户端正则。

**杀手级展示实验（强烈建议做）：**
- 人为只给**指定的某些客户端**注入高噪声，画出 agent 每轮分配的权重热力图，
  证明它**准确识别并降权了被污染的客户端**。这张图比任何指标都有说服力。

---

## 8. 诚实的风险与工作量

- **不保证一定赢**：agent 可能判断不准。但即便打平，"自适应 agent vs 静态 robust"的对比本身比纯基准有看点。
- **工作量**：策略 A 约 1–2 周（含实验）；策略 B 额外几天；杀手级展示实验 1–2 天。
- **probe 集**会引入一点额外设定，要在论文里交代清楚（联邦里常见，可接受）。
- LLM 编排的可复现性需固定随机性/温度，并以 A 为定量主结论。

## 9. 分阶段计划

1. **P0（地板，进行中）**：跑完现有基准 + ODIR，确立 FedAvg/robust 基线与鲁棒性研究。
2. **P1（创新核心）**：实现策略 A 编排器 + `agent_fed`，跑主对比 + 消融。
3. **P2（杀手实验）**：定向注入噪声 + 权重识别热力图。
4. **P3（叙事/可解释）**：可选 LLM 编排层 + 跨数据集复现。
5. **P4**：按 `docs/PAPER_OUTLINE.md` 把 agent 作为主贡献重排成稿。

---

## 10. 是否值得做（决策）

- 若 P0 的噪声/非IID 显示**静态 robust 已能赢 FedAvg** → agent 是"锦上添花",论文也能中端;
- 若 P0 显示**没有静态方法能赢 FedAvg**（按 clean 趋势，概率不低）→ **agent 是唯一能产生真创新的路径**，强烈建议做。

无论哪种，agent 都是从"够发"走向"值得发"的关键一步。
